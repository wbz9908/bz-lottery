# bz-lottery 技术澄清技能

## 使用场景

当任务存在以下情况之一时，先使用本技能：

- 需求描述不完整，存在关键业务或技术问题待确认。
- 涉及后端、前端、部署、数据库等多个区域。
- 涉及接口契约、数据结构、权限、并发、幂等或兼容性。
- 用户要求先产出方案、设计、技术澄清或开发计划。

简单修复、明确的小改动可以直接进入开发流程，不必强制生成澄清文档。

## 工作流

1. 阅读用户需求和相关现有代码/文档。
2. 明确本次改动的目标、非目标和边界。
3. 按影响范围梳理后端、前端、部署、数据、接口和测试点。
4. 标记必须向用户确认的问题；如果没有阻塞问题，给出合理假设并继续。
5. 形成技术澄清文档或在对话中输出同等结构的澄清结果。

## 输出结构

```markdown
# 技术澄清文档

## 背景

## 目标

## 非目标

## 影响范围

## 方案设计

## 接口与数据变化

## 验证方案

## 风险与回滚

## 待确认问题
```

## 规则

1. 不把不确定事项写成已确认事实。
2. 涉及数据库变更时，必须说明 Flyway 迁移和兼容策略。
3. 涉及接口变化时，必须说明调用方影响和文档更新点。
4. 涉及并发、库存、支付、抽奖结果等关键链路时，必须说明幂等和一致性策略。
5. 澄清文档是开发输入，不替代最终验证。

## 交付要求

交付内容包含：

- 需求理解
- 改动范围
- 推荐方案
- 验证方案
- 风险
- 需要用户确认的问题
